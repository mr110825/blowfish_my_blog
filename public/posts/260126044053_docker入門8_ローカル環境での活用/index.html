この記事でやること
#

日常の開発・学習で使える実践的なDockerの活用パターンを紹介します。

使い捨て学習環境のパターン
#

練習用のコンテナを気軽に作成・削除できるのがDockerの強みです。

パターン1：単発のコマンド練習
# 

# Ubuntuコンテナでコマンド練習（終了後自動削除）
docker run --rm -it ubuntu bash

# Pythonの動作確認
docker run --rm -it python:3.12 python

# Node.jsのREPL
docker run --rm -it node:20 node

--rmオプションにより、コンテナ終了時に自動削除されます。

パターン2：一時的なデータベース
# 

# 練習用MySQL（停止時にデータ削除）
docker run --rm -it \
  -e MYSQL_ROOT_PASSWORD=test \
  -e MYSQL_DATABASE=practice \
  -p 3306:3306 \
  mysql:8.0

# PostgreSQL
docker run --rm -it \
  -e POSTGRES_PASSWORD=test \
  -p 5432:5432 \
  postgres:16

# Redis
docker run --rm -it \
  -p 6379:6379 \
  redis:7

パターン3：複数構成の練習環境
# 

# practice/docker-compose.yml
services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: practice
      MYSQL_DATABASE: test

cd practice
docker compose up -d    # 起動
docker compose down -v  # 完全削除

よく使う開発パターン
#

フロントエンド開発
# 

services:
  node:
    image: node:20-alpine
    working_dir: /app
    volumes:
      - .:/app
    ports:
      - "3000:3000"
    command: npm run dev

Python開発
# 

services:
  python:
    image: python:3.12
    working_dir: /app
    volumes:
      - .:/app
    command: python app.py

APIサーバー + データベース
# 

services:
  api:
    build: .
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgres://postgres:password@db:5432/app
    depends_on:
      - db

  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: password
      POSTGRES_DB: app
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:

クリーンアップのコマンド
#

練習していると不要なリソースが溜まりやすいです。定期的にクリーンアップを行います。

# 停止中のコンテナを削除
docker container prune

# 使用していないイメージを削除
docker image prune

# 使用していないボリュームを削除
docker volume prune

# 使用していないネットワークを削除
docker network prune

# まとめて削除（注意して実行）
docker system prune -a

docker system prune -aは使用していないイメージもすべて削除するため、必要なイメージまで消える可能性があります。注意して実行してください。
