この記事でやること
#

Docker Composeを使って、WordPress + MySQL環境を1コマンドで構築します。

Docker Composeとは
#

Docker Composeは、複数のコンテナを一括で定義・管理するためのツールです。

これまではdocker runコマンドを何度も実行してコンテナを起動していました。Docker Composeを使うと、docker-compose.yml（またはcompose.yaml）に構成を記述し、1コマンドで複数コンテナを起動できます。

なぜDocker Composeを使うのか
#

従来の方法（複数のdocker runコマンド）
# 

# ネットワーク作成
docker network create wp-net

# MySQL起動
docker run -d \
  --name mysql \
  --network wp-net \
  -e MYSQL_ROOT_PASSWORD=secret \
  -e MYSQL_DATABASE=wordpress \
  -v mysql-data:/var/lib/mysql \
  mysql:8.0

# WordPress起動
docker run -d \
  --name wordpress \
  --network wp-net \
  -p 8080:80 \
  -e WORDPRESS_DB_HOST=mysql \
  -e WORDPRESS_DB_PASSWORD=secret \
  wordpress:latest

問題点：

  • コマンドが長い
  • 起動順序を覚えておく必要がある
  • 設定がドキュメント化されない

Docker Composeを使う方法
# 

# docker-compose.yml に構成を記述
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_DATABASE: wordpress
    volumes:
      - mysql-data:/var/lib/mysql

  wordpress:
    image: wordpress:latest
    ports:
      - "8080:80"
    environment:
      WORDPRESS_DB_HOST: mysql
      WORDPRESS_DB_PASSWORD: secret
    depends_on:
      - mysql

volumes:
  mysql-data:

# 1コマンドで起動
docker compose up -d

docker-compose.ymlの構造
# 

services:        # コンテナの定義
  <サービス名>:
    image: ...   # 使用するイメージ
    build: ...   # Dockerfileからビルドする場合
    ports: ...   # ポートマッピング
    volumes: ... # ボリュームマウント
    environment: # 環境変数
    depends_on:  # 依存関係

volumes:         # 名前付きボリュームの定義
  <ボリューム名>:

networks:        # ネットワークの定義（省略可）
  <ネットワーク名>:

主要な設定項目
#

項目 説明 例
image 使用するイメージ mysql:8.0
build Dockerfileのパス ./app
ports ポートマッピング "8080:80"
volumes ボリュームマウント mysql-data:/var/lib/mysql
environment 環境変数 MYSQL_ROOT_PASSWORD: secret
depends_on 起動順序の依存関係 - mysql
restart 再起動ポリシー always

Docker Composeコマンド
#

コマンド 説明
docker compose up コンテナを作成・起動
docker compose up -d バックグラウンドで起動
docker compose down コンテナを停止・削除
docker compose ps コンテナの状態を表示
docker compose logs ログを表示
docker compose logs -f ログをリアルタイム表示
docker compose exec <service> bash コンテナに接続
docker compose build イメージをビルド

ハンズオン：WordPress + MySQL環境を構築
#

Docker Composeを使って、WordPressとMySQLの環境を構築します。

1. 作業ディレクトリの作成
# 

mkdir ~/wordpress && cd ~/wordpress

2. docker-compose.ymlを作成
# 

cat <<EOF > docker-compose.yml
services:
  db:
    image: mysql:8.0
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wordpress
      MYSQL_PASSWORD: wordpress
    volumes:
      - db-data:/var/lib/mysql

  wordpress:
    image: wordpress:latest
    restart: always
    ports:
      - "8080:80"
    environment:
      WORDPRESS_DB_HOST: db
      WORDPRESS_DB_USER: wordpress
      WORDPRESS_DB_PASSWORD: wordpress
      WORDPRESS_DB_NAME: wordpress
    depends_on:
      - db
    volumes:
      - wp-data:/var/www/html

volumes:
  db-data:
  wp-data:
EOF

3. コンテナを起動
# 

docker compose up -d

出力例：

[+] Running 3/3
 ✔ Network wordpress_default  Created
 ✔ Container wordpress-db-1   Started
 ✔ Container wordpress-wordpress-1  Started

4. 状態確認
# 

docker compose ps

5. WordPressにアクセス
#

ブラウザでアクセスします。

http://localhost:8080

WordPressの初期設定画面が表示されれば成功です。

6. ログの確認
# 

# 全サービスのログ
docker compose logs

# 特定サービスのログ
docker compose logs wordpress

7. コンテナに接続
# 

# WordPressコンテナに接続
docker compose exec wordpress bash

# MySQLに接続
docker compose exec db mysql -uwordpress -pwordpress wordpress

8. 停止・削除
# 

# コンテナを停止・削除（ボリュームは保持）
docker compose down

# ボリュームも含めて完全削除
docker compose down -v

depends_onの注意点
#

depends_onは起動順序を制御しますが、サービスが準備完了するまで待つわけではありません。

例えば、MySQLコンテナが起動しても、データベースが接続を受け付ける状態になるまでに時間がかかります。その間にWordPressが接続しようとすると失敗する可能性があります。

対策：

  • アプリケーション側でリトライ処理を実装
  • ヘルスチェックを設定（healthcheck）
services:
  db:
    image: mysql:8.0
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 5s
      timeout: 3s
      retries: 10

  wordpress:
    depends_on:
      db:
        condition: service_healthy

環境変数の管理
#

機密情報はdocker-compose.ymlに直接書かず、.envファイルを使います。

.envファイルを作成
# 

# .env
MYSQL_ROOT_PASSWORD=rootpassword
MYSQL_PASSWORD=wordpress

docker-compose.ymlで参照
# 

services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}

まとめ
#

  • Docker Composeで複数コンテナを一括管理できる
  • docker-compose.ymlに構成を記述
  • docker compose up -dで起動、docker compose downで停止
  • depends_onは起動順序の制御（準備完了は待たない）
  • 機密情報は.envファイルで管理
