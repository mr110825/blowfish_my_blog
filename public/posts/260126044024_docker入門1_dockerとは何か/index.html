Dockerとは何か
#

Dockerは、アプリケーションをコンテナという単位でパッケージ化し、どの環境でも同じように動作させるためのプラットフォームです。

従来、開発環境では動くのに本番環境では動かない、という問題がよく発生していました。原因は、OSのバージョン、ライブラリ、設定ファイルなど、環境ごとの微妙な差異にあります。

Dockerは、アプリケーションとその実行に必要な依存関係をすべてコンテナに含めることで、この問題を解決します。

観点 従来の課題 Dockerによる解決
環境差異 「自分のPCでは動く」問題 コンテナ内に依存関係を含める
セットアップ 手順書どおりにやっても動かない Imageから即座に同一環境を再現
配布 環境構築に時間がかかる Imageを共有するだけ

仮想化技術の種類
#

仮想化技術は大きく2種類に分けられます。

ハイパーバイザー型（従来の仮想マシン）
#

ハードウェア上にハイパーバイザーを配置し、その上でゲストOSを動作させる方式です。

代表的なツール：VMware、VirtualBox、Hyper-V

  • ゲストOSごとにカーネルを持つ
  • 完全な分離性を持つ
  • 起動に時間がかかる（数分）
  • リソース消費が大きい

コンテナ型（Docker）
#

ホストOSのカーネルを共有し、プロセスレベルで隔離する方式です。

代表的なツール：Docker、Podman、containerd

  • ホストOSのカーネルを共有
  • 軽量で高速（起動は数秒）
  • リソース効率が良い
  • Linuxカーネルの機能（namespace、cgroups）で隔離を実現

比較表
#

観点 ハイパーバイザー型 コンテナ型
起動時間 数分 数秒
リソース消費 大（GB単位） 小（MB単位）
分離レベル 完全（OS単位） プロセス単位
可搬性 低い 高い
代表例 VMware, VirtualBox Docker, Podman

Dockerを使うメリット
#

1. 環境の統一
#

開発環境・テスト環境・本番環境で同じコンテナを使用できます。「開発環境では動くのに本番で動かない」という問題を防げます。

2. 軽量・高速
#

仮想マシンと比べて起動が速く（数秒）、リソース消費も少ないです。1台のホストで多数のコンテナを動作させることができます。

3. 再現性
#

Dockerfileやdocker-compose.ymlに環境構成をコードとして記述できます。これにより、誰でも同じ環境を再現できます。

4. 可搬性
#

コンテナイメージを共有すれば、どの環境でも同じように動作します。Docker Hubを通じてイメージを配布することも容易です。

5. 隔離性
#

コンテナ同士は独立しており、互いに影響を与えません。異なるバージョンのライブラリを同時に動作させることも可能です。

基本用語の整理
#

Docker Image（イメージ）
#

コンテナの元となるテンプレート。アプリケーションと依存関係がパッケージ化されています。読み取り専用。

Docker Container（コンテナ）
#

Imageから作成された実行環境。Imageを元に起動し、停止・削除が可能。同じImageから複数のコンテナを作成できます。

Docker Hub
#

Imageを共有・配布するためのレジストリサービス。公式イメージ（nginx、mysql、nodeなど）が公開されています。

Docker Engine
#

Dockerの中核となるソフトウェア。コンテナの作成・実行・管理を行います。Docker daemonとDocker CLIで構成されます。

ImageとContainerの関係
#

概念 例え 特徴
Image 設計図・金型 読み取り専用、変更不可
Container 製品・実体 Imageから作成、実行・停止・削除が可能

まとめ
#

  • Dockerはアプリケーションをコンテナとしてパッケージ化し、環境差異の問題を解決する
  • コンテナ型仮想化は、ハイパーバイザー型と比べて軽量・高速
  • ImageはコンテナのテンプレートでContainerはその実行環境
  • Docker Hubで公式イメージを取得できる
