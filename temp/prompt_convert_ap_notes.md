# 応用情報技術者試験解説メモ → スクラップ記事変換プロンプト

## 変換の流れ

この変換プロセスは以下の4ステップで構成されます：

1. **準備作業**：ディレクトリ作成とファイル準備
2. **記事作成**：構造化された記事の生成
3. **品質チェック**：内容とフォーマットの確認
4. **公開準備**：インデックス更新とビルド確認

## 変換指示

以下の手順で詳細な解説メモを一般読者向けのスクラップ記事に変換してください：

### STEP 1: 準備作業

#### 1-1. ディレクトリとファイル作成
**必須作業：**以下を順番に実行してください

```bash
# 1. ディレクトリ名を決定（形式: ap_rXX_season_qXX_テーマ）
# 例: ap_r05_autumn_q01_infosec, ap_r07_spring_q01_database

# 2. Hugoでファイル作成
hugo new scraps/[ディレクトリ名]/index.md

# 3. 作成されたファイルの確認
ls -la content/scraps/[ディレクトリ名]/
```

**命名規則：**
- `rXX`: 令和の年（05, 06, 07等）
- `season`: spring（春期）またはautumn（秋期）
- `qXX`: 問題番号（01, 02等）
- `テーマ`: infosec, database, network, program等

#### 1-2. Front Matter設定
**テンプレート：**
```toml
+++
date = '[自動生成される現在日時]'
draft = false
title = '応用情報技術者試験 R[XX][時期]Q[XX]（[テーマ名]）解説'
tags = ["応用情報技術者試験", "[メインテーマ]", "[サブテーマ1]", "[サブテーマ2]"]
+++
```

**タグ選択ガイド：**
| テーマ | 推奨タグ |
|:---|:---|
| **情報セキュリティ** | "情報セキュリティ", "サイバー攻撃", "脆弱性" |
| **データベース** | "データベース", "SQL", "正規化" |
| **ネットワーク** | "ネットワーク", "TCP/IP", "ルーティング" |
| **プログラミング** | "プログラミング", "アルゴリズム", "データ構造" |
| **システム開発** | "システム開発", "要件定義", "テスト" |

### STEP 2: 記事構成の作成

#### 2-1. 推奨記事構成

```markdown
## はじめに
[3-4文で問題の概要と重要性を説明]
[学習価値を明確に提示]

## 問題の全体像
### 登場人物と関係性
| 組織名 | 役割 | 詳細 |
|:---|:---|:---|

### システム構成の理解
[図表の重要ポイントを文章で補足]

### 攻撃/処理シナリオの時系列
1. **段階1**: [何が起こったか]
2. **段階2**: [次に何が起こったか]
...

---

## 設問別解説

### 設問X: [設問のテーマ]

**問題**: [設問文の要約]

**解答**: [簡潔な解答]

**解説**:
[思考プロセスを段階的に説明]

**重要ポイント**:
- [学習すべき点1]
- [学習すべき点2]

---

## まとめ
[3-5の重要ポイント]

## 参考リンク
- [関連技術の公式サイト等]
```

#### 2-2. 文体・表現の統一ルール
- **丁寧語を使用** - 読者に親しみやすい文体
- **専門用語には適切な説明を併記** - 理解しやすさを重視
- **読みやすい段落構成** - 適切な改行と空行
- **重要箇所は太字で強調** - `**重要**`で囲む
- **コードや技術用語はバッククォート** - `技術用語`で囲む
- **表形式の積極活用** - 比較や一覧は表で整理
- **見出し階層の適切な使用** - H2, H3, H4を体系的に使用

### STEP 3: 品質チェック

#### 3-1. 自動チェックポイント
**実行前チェック：**
```bash
# ファイルの存在確認
ls -la content/scraps/[ディレクトリ名]/index.md

# 文法チェック（markdownlint等があれば）
# シンタックスエラーの確認
```

#### 3-2. 内容品質チェックリスト
**必須チェック項目：**
- [ ] Front matterの日付が正確
- [ ] タイトルが「応用情報技術者試験 R[XX][時期]Q[XX]（[テーマ]）解説」形式
- [ ] tagsに「応用情報技術者試験」が含まれている
- [ ] 表組みが適切に使用されている（最低1つ）
- [ ] 専門用語に適切な説明がある
- [ ] 各設問に「解答」「解説」が明記されている
- [ ] 見出し階層が適切（H2→H3→H4の順）
- [ ] 読者の学習に役立つ内容になっている

**推奨チェック項目：**
- [ ] コードや技術用語がバッククォートで囲まれている
- [ ] 重要な部分が太字で強調されている
- [ ] 適切な改行と空行で読みやすい
- [ ] 既存記事と同等のクオリティ

### STEP 4: 公開準備

#### 4-1. インデックスページの更新（必須）
**手順：**
```bash
# 1. インデックスファイルを開く
vim content/scraps/_index.md  # または適切なエディタ

# 2. 応用情報技術者試験セクションを見つける
# 3. 新しい記事リンクを最上位に追加
```

**リンク形式：**
```markdown
- [応用情報技術者試験 R[XX][時期]Q[XX]（[テーマ]）解説]({{< ref "[ディレクトリ名]" >}})
```

#### 4-2. 最終確認とビルド
**実行コマンド：**
```bash
# サイトのビルドテスト
hugo --buildDrafts

# エラーがないことを確認
echo $?  # 0 が返されればOK

# ローカルサーバーで表示確認（オプション）
hugo server -D
```

## エラー対処法とトラブルシューティング

### よくある問題と解決法

| 問題 | 原因 | 解決法 |
|:---|:---|:---|
| **Hugo newが失敗** | ディレクトリ名に使用できない文字 | 英数字とハイフンのみ使用 |
| **ビルドエラー** | Front matterの文法エラー | TOMLの構文を確認 |
| **表示されない** | draft = true のまま | draft = false に変更 |
| **リンクが無効** | インデックス更新忘れ | _index.md に手動追加 |

### 記事作成後の確認コマンド
```bash
# ファイル構造の確認
tree content/scraps/[ディレクトリ名]/

# 記事のプレビュー（最初の20行）
head -20 content/scraps/[ディレクトリ名]/index.md

# Hugo設定の検証
hugo config | grep -i baseurl
```

## 自動化のためのヒント

### 効率化のポイント
1. **テンプレートの活用**: Front matterとヘッダー構造は使い回し可能
2. **バッチ処理**: 複数ファイルの一括作成・更新
3. **検証スクリプト**: 品質チェックの自動化

### 次回作業時の改善
- 前回作業で気づいた改善点をメモ
- 新しい問題パターンに対応した構成の検討
- タグ体系の継続的な整理

## 使用例とベストプラクティス

**このプロンプトの使用法：**
1. temp配下の解説メモを用意
2. このプロンプトを参考に段階的に変換作業を実施
3. 既存記事（`content/scraps/ap_r06_autumn_q01_infosec/index.md`等）の品質を参考基準とする
4. 変換完了後、このプロンプト自体の改善点があれば記録

**品質向上のコツ：**
- 読者目線での分かりやすさを最優先
- 試験対策と実務知識の両方を意識
- 継続的な記事品質の向上
